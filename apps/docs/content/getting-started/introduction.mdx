# Introduction

**Harish UI** is a professional React component library built for real applications.

## Why Harish UI?

Most component libraries force trade-offs: either you get beautiful defaults that are hard to customize, or you get headless primitives that require you to write all the CSS yourself.

Harish UI takes a third approach:

- **CSS variables for theming** — one import, override tokens to match your brand. No special setup needed.
- **TypeScript-first** — every component, variant, and prop is fully typed and exported.
- **Accessible by default** — ARIA roles, keyboard navigation, and focus management built into every component from day one.
- **SSR-ready** — works in Next.js App Router, Remix, and any React environment without extra configuration.
- **Composable** — the `asChild` prop lets you render any component as any HTML element.

## Design philosophy

> Simple APIs. Predictable behavior. Escape hatches where you need them.

Components are designed to work without configuration, but expose every necessary hook for customization. You should be productive in 30 seconds and never hit a ceiling.

## What's included

- **9 Phase 1 components** ready for production: Button, Input, Textarea, Label, Badge, Card, Stack, Divider, Spinner
- **ThemeProvider** with light/dark/system modes
- **CSS variable tokens** for complete theme control
- **Storybook** for component development and visual testing
- **Changesets** for automated versioning and npm publishing
